# 4월 정산상세 대표 원형 감리

- 작성일: 2026-05-07
- 대상 루트: `\\172.16.10.120\소설사업부\판무팀_ssot\100_계산서_매출등록_자료`
- 범위: 각 플랫폼의 `4월` 폴더 아래 파일명에 `정산상세`가 포함된 원형 파일. 원본 파일은 읽기 전용으로만 확인했다.
- 목적: 플랫폼별 초기 원형을 `S2 전송자료` 생성 파이프라인에 넣을 수 있는지 적대적으로 판정한다.

## 결론

- 37개 플랫폼 중 4월 대표 원형이 있는 플랫폼은 16개, 없는 플랫폼은 21개다.
- 4월 원형이 있다고 해서 곧바로 “헤더만 통일하면 됨”은 아니다. 일반 어댑터 후보는 8개뿐이고, 8개는 금액 의미 확인 또는 특수 파서가 필요하다.
- 한아름은 사용자가 본 느낌이 맞다. 한 시트 안에 요약, 시간 로그, 건당 로그, 계산서 안내, 반복 헤더, 합계가 섞여 있어 일반적인 `첫 헤더 row + 데이터 body` 방식으로는 깨진다.
- 윌라는 4월 폴더 자체가 없어 이번 4월 대표 감리에는 들어오지 않는다. 다만 이전 전수조사 기준으로는 `sheet`와 `콘텐츠 가격 변동 이력` 2시트 구조가 맞으므로, 윌라는 fallback 월로 별도 대표 감리를 해야 한다.

## 감리 3회

1. 파일 감리: `4월` 폴더 존재, `정산상세` 파일 존재, 복사본/통합본/선투자/보조 시트 같은 파일 위생 신호를 확인했다.
2. 스키마 감리: 시트별 행/열 크기, 병합 셀, 헤더 후보 row, 제목/ID/판매/정산/상계 후보 컬럼을 확인했다.
3. S2 적합성 감리: 최종 산출물 `판매채널콘텐츠ID / 판매금액 / 정산기준액 / 상계금액`을 만들 수 있는지 기준으로 일반 어댑터, 금액 규칙 확인, 특수 파서, 4월 샘플 없음으로 재분류했다.
