## 적대적 감리

### 1차: `정산상세` 전수조사만으로 표준 스키마를 바로 확정할 수 있는가

아직 아니다. 헤더 후보와 컬럼명 후보는 충분히 확보했지만, 금액 컬럼의 의미가 판매처마다 다르다. 예를 들어 `정산금액`, `공급가액`, `입금액`, `정산액`이 모두 S2의 `정산기준액`으로 들어가야 하는지, 아니면 일부는 세전/세후/수수료 차감 후인지 확인이 필요하다.

### 2차: 제목 컬럼 후보를 keyword로 자동 선택해도 되는가

안 된다. `컨텐츠No`, `작품코드`, `채널상품ID`처럼 제목처럼 보이지만 실제로는 ID인 컬럼이 있다. 제목/ID/금액 컬럼 선택은 판매처별 schema adapter가 가져야 한다.

### 3차: KISS/S2 API만 붙이면 수동 루틴을 대체할 수 있는가

아니다. KISS 읽기 API는 `판매채널콘텐츠ID` 후보를 제공할 수 있지만, 원본 정산상세 금액을 S2 4컬럼으로 변환하는 규칙은 별도다. API 연동보다 먼저 정산상세 표준화 adapter가 필요하다.

## 구현 방향

1. `SettlementSchemaAdapter` 개념을 둔다.
2. adapter는 헤더 시그니처/폴더/파일명으로 파일 타입을 감지한다.
3. adapter별로 다음 필드를 표준 스키마로 매핑한다.
   - `raw_title`
   - `external_content_id`
   - `sale_amount`
   - `settlement_base_amount`
   - `offset_amount`
4. KISS/S2 읽기 API로 `schnCtnsId`, `ctnsId`, `ctnsNm`, `schnId`, `schnNm` 후보 목록을 가져온다.
5. 표준 정산상세와 KISS/S2 후보를 매칭한다.
6. `matched`만 `S2전송자료`로 내보낸다.
7. `no_match`, `ambiguous`, `schema_error`는 검토 시트로 분리한다.
