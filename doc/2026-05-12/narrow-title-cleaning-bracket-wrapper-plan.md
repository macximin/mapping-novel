# 좁은 텍스트 클리닝 고도화: 대괄호 제목 wrapper 복구

작성일: 2026-05-12

## 결론

텍스트 클리닝 고도화는 넓게 바꾸지 않는다.

1차 수정 범위는 **정산서 콘텐츠명이 대괄호 하나로만 감싸진 제목인 경우 내부 제목을 살리는 것**으로 제한한다.

```text
[효종] -> 효종
```

이번 수정은 `blank_key` 방지 목적이다. 기존 `no_match` 전체를 한 번에 해소하려는 작업이 아니다.

## 근거 파일

기준 파일:

```text
C:\Users\wjjo\Desktop\mapping-novel\텍스트 클리닝 고도화.xlsx
```

시트:

```text
전체_행별매핑_종합 (2)
```

확인 결과:

```text
전체 행: 82,957
matched: 75,095
no_match: 7,861
blank_key: 1
```

`blank_key` 1건:

```text
파일: 2026-02__문피아__문피아__2026년 2월 문피아 정산상세.xlsx
S2 판매채널: 문피아
정산서_콘텐츠명: [효종]
현재 정제_상품명: 빈 값
현재 S2_매칭상태: blank_key
```

현재 `clean_title("[효종]")`은 빈 문자열을 반환한다. 원인은 `clean_title()`이 `[...]` 대괄호 내용을 전부 메타 표식으로 보고 삭제하기 때문이다.

## S2 기준 확인

로컬 S2 lookup 기준으로 `효종`은 문피아 지급정산 기준에 존재한다.

```text
판매채널명: 문피아
콘텐츠명: 효종
판매채널콘텐츠ID: 863933
콘텐츠ID: 116477
지급정산상태: 운영중
승인상태: 승인
```

또한 문피아에는 별도로 `(연재)효종`도 존재한다.

```text
판매채널명: 문피아
콘텐츠명: (연재)효종
판매채널콘텐츠ID: 165047
콘텐츠ID: 107852
지급정산상태: 운영중
승인상태: 승인
```

따라서 괄호 정제를 넓게 바꾸면 `효종`과 `(연재)효종`의 분리가 흔들릴 수 있다.

## 1차 규칙

다음 조건을 모두 만족할 때만 대괄호 내부를 제목으로 사용한다.

1. 원문 전체가 `[...]` 형태 하나뿐이다.
2. 내부 문자열이 비어 있지 않다.
3. 내부 문자열이 차단 표식이 아니다.
   - `사용안함`
   - `사용금지`
   - `정산정보없음`
4. 내부 문자열이 단순 장르/유형 태그로만 보이지 않는다.
   - 예: `BL`, `GL`, `TL`, `연재`, `단행본`, `외전`, `완결`

허용 예:

```text
[효종] -> 효종
```

기존 유지 예:

```text
[BL] 효종 -> 효종
[연재]효종 -> 효종
효종 1화 -> 효종
```

차단 유지 예:

```text
[사용안함] 삭제 작품
[사용금지] 삭제 작품
[정산정보없음] 삭제 작품
[사용안함]_[정산정보없음] 이중 차단 작품
```

## 하지 않는 것

이번 1차 수정에서는 하지 않는다.

- 소괄호 wrapper 전체 복구
- `(연재)효종`의 정제 방식 변경
- 모든 `[...]` 내용을 제목으로 보존
- `no_match` 7,861건 전체를 대상으로 한 광범위 클리닝
- S2 매칭 fallback 추가
- 제목 유사도 매칭 추가
- 판매채널별 예외 규칙 대량 추가

## 구현 위치

대상 파일:

```text
cleaning_rules.py
```

대상 함수:

```text
CleaningPolicy.clean_title()
```

현재 문제 구간:

```python
t = re.sub(r"\([^)]*\)|\[[^\]]*\]", "", t)
```

권장 구현은 이 줄을 넓게 바꾸는 것이 아니라, 이 줄보다 앞에서 **전체 대괄호 wrapper 제목만 좁게 복구**하는 helper를 두는 방식이다.

개념:

```python
wrapper = self.unwrap_square_title_if_safe(t)
if wrapper:
    t = wrapper
else:
    t = re.sub(r"\([^)]*\)|\[[^\]]*\]", "", t)
```

## 적대적 감리

### 1. 대괄호 태그 오인 위험

공격:

`[BL]`, `[연재]`, `[외전]` 같은 태그를 제목으로 살리면 기존에 잘 매칭되던 제목이 깨질 수 있다.

보정:

대괄호 wrapper가 원문 전체일 때만 복구한다. 원문 뒤에 제목 본문이 이어지는 `[BL] 효종`은 기존처럼 태그 제거 후 `효종`으로 둔다.

판정:

복구 조건을 `fullmatch(r"\[[^\]]+\]")`로 제한하면 위험이 낮다.

### 2. 차단 표식 무력화 위험

공격:

`[정산정보없음]` 같은 차단 표식을 내부 제목으로 살리면, 기존 선차단 정책이 깨질 수 있다.

보정:

차단 표식은 `drop_disabled_rows()`가 먼저 전 행을 제거하는 정책을 유지한다. 또한 wrapper 복구 helper에서도 내부 문자열이 차단 표식이면 복구하지 않는다.

판정:

차단 표식 회귀 테스트를 반드시 유지한다.

### 3. 효종 / (연재)효종 혼선 위험

공격:

S2에 `효종`과 `(연재)효종`이 모두 존재한다. 괄호 정제를 넓게 바꾸면 두 작품이 같은 정제키로 섞일 수 있다.

보정:

이번 수정은 대괄호 wrapper 하나만 대상으로 한다. 소괄호 정제와 `(연재)` 처리 방식은 손대지 않는다.

판정:

`[효종]` 1건을 살리되, 소괄호 관련 정책은 별도 2차 검토로 분리한다.

진행 가능.
