# S2 수동 승인 alias 패치 실제 배치 검증

작성일: 2026-05-12

## 처리한 항목

사용자 승인에 따라 자동 일반 규칙이 아니라 exact alias로만 처리했다.

| 기존 정제키 | 신규 정제키 | 사유 |
| --- | --- | --- |
| `늙은경비에게조교당하는스튜어디스의이야기` | `늙은경비에게조교당하는스튜어디스` | S2 제목으로 통일 |
| `늙은경비에게조교당하는스튜어디스이야기` | `늙은경비에게조교당하는스튜어디스` | S2 제목으로 통일 |
| `천대받는f급힐러라좋았는데요` | `천대받는f급힐러라서좋았는데요` | S2 제목으로 통일 |

`이야기` 일반 제거 규칙은 추가하지 않았다.

## 추가 다중 병렬 Survey 2: 전체 판단 파티션

업무 판단까지 포함해 최신 잔여 `no_match` 전체를 다시 닫아 봤다.

기준:

- 같은 채널에 fuzzy 후보가 있는지
- 다른 채널에 exact 지급정산이 있는지
- 다른 채널에 fuzzy 근접 후보만 있는지
- S2 지급정산 lookup 전체에 같은 정제키가 아예 없는지

결과: 잔여 484그룹 / 5,807행 전부를 아래 5개 판단 영역으로 분류했다.

| 같은채널 fuzzy 작품동일성 판단 | 3 | 4 | 같은 판매채널 안에 유사 제목 지급정산이 있음 | 동일작이면 exact alias, 아니면 보류 |

- 즉시 일반 로직으로 넓힐 후보: 없음.

- S2 채널/지급정산 보강 후보: `채널 보강 판단` 10그룹 / 30행 + `타채널 exact 지급정산 보강 판단` 232그룹 / 2,271행.
- S2 신규 생성 또는 no-action 판단 후보: 236그룹 / 3,488행.

같은채널 fuzzy 후보는 작지만, 로직으로 넓히기에는 위험하다.

## S2 미매핑 사유 고도화

범위는 정보 제공으로 제한했다. 매칭 상태를 바꾸거나 S2 지급정산/판매채널콘텐츠를 자동 생성하지 않는다.

행별 종합 리포트에 아래 컬럼을 추가했다.

| 컬럼 | 내용 |
| --- | --- |
| `S2_미매핑상세사유` | no_match/blank/ambiguous의 세부 원인 |
| `S2_미매핑근거` | 후보 ID, 콘텐츠ID, 판매채널, 콘텐츠마스터 근거 |
| `S2_권장조치` | PD 작업지시용 다음 확인 방향 |

현재 구분하는 사유:

- 해당채널 판매채널콘텐츠 있음 / 지급정산 없음
- 청구정산 후보 있음
- 해당채널 지급정산 없음 / 타채널 지급정산 존재
- 콘텐츠마스터 있음 / S2 지급정산 없음
- S2/콘텐츠마스터 정제키 후보 없음
- 정제키 없음
- S2 중복 후보

PD 작업지시 리포트에는 `S2_미매핑상세사유`, `S2_미매핑근거`를 노출하고, `권장액션`은 `S2_권장조치`가 있으면 그것을 우선 사용한다.

추가로 현재 판매채널 S2 기준표가 비어 있는 경우에도 배치가 죽지 않고 `no_match` 행과 사유 컬럼을 만들도록 보강했다.
