# S2 no_match 좁은 판단 영역 설계

작성일: 2026-05-12

## 입력 기준

- 검토 파일: `텍스트 클리닝 고도화.xlsx`
- 시트: `전체_행별매핑_종합 (2)`
- 비교 S2: `data/kiss_payment_settlement_s2_lookup.csv`
- 비교 단위: `S2 판매채널 + 정제_상품명`

## 전체 규모

| 기준 | 수 |
| --- | ---: |
| 검토필요 행 | 7,862 |
| `판매채널 + 원문명` 중복 제거 | 4,522 |
| `판매채널 + 정제키` 중복 제거 | 551 |

행 단위로 보면 너무 크지만, 판단 단위는 `판매채널 + 정제키` 551개까지 줄어든다. 이후 작업은 이 551개를 대상으로 한다.

## 1차 버킷

| 버킷 | 그룹 수 | 행 수 | 판단 |
| --- | ---: | ---: | --- |
| S2 어느 채널에도 현재 정제키 없음 | 315 | 5,571 | 로직 후보와 S2 데이터 누락이 섞여 있음. 2차 후보 탐색 필요 |
| S2에는 있으나 다른 채널에만 있음 | 235 | 2,290 | 기본적으로 판매채널 지급정산 없음/채널 기준 차이. 로직 수정 우선순위 낮음 |
| 빈 정제키 | 1 | 1 | `[효종]` 케이스. 매우 좁은 클리닝 후보 |

## 귀축기담 판단

`귀축기담`은 S2 지급정산에 존재한다.

확인된 S2 채널:

- 밀리의 서재
- 알라딘(소설)(1068659313)
- 네이버_연재
- 윌라(전자책)
- 판무림
- 한아름(P000003716)
- 원스토어(소설)
- 미스터블루(소설)
- 웹소설 하이북
- 블라이스_일반결제
- 미소설
- 무툰(소설 서비스)
- 리디북스(소설)

하지만 `Yes24(서점)` 지급정산 행은 없다. 따라서 예스24 정산서의 `귀축기담 01권` 등은 현재 기준으로는 로직 문제가 아니라 `Yes24(서점)` 지급정산 기준 데이터 없음으로 보는 쪽이 맞다.

## 즉시 로직 후보

### 1. 대괄호 전체 래핑

현재:

- `[효종]` -> 빈 키
- S2 문피아에는 `효종` 존재

판단:

- 대괄호 안이 전체 제목인 경우만 제한적으로 살리는 규칙 후보.
- `[BL] 효종`, `[연재]효종` 같은 접두 태그형은 기존 흐름 유지가 필요하다.

### 2. 제목 말미 `제` 손실

현재 정제 예:

- `파공검제 30권` -> `파공검`
- `마교육제 242화` -> `마교육`
- `수라검제 2권` -> `수라검`

같은 채널 S2에는 각각 `파공검제`, `마교육제`, `수라검제`가 있다.

확인된 유효 후보:

| 채널 | 현재 정제키 | 행 수 | 같은 채널 S2 후보 |
| --- | --- | ---: | --- |
| 리디북스(소설) | 파공검 | 686 | 파공검제 |
| 리디북스(소설) | 마교육 | 179 | 마교육제 |
| 리디북스(소설) | 수라검 | 35 | 수라검제 |
| 구글(소설) | 파공검 | 7 | 파공검제 |

주의할 오탐:

- `검(劍)` -> `검`인데 S2에 `검제(전2부작)`이 있다고 해서 매핑하면 안 된다.

따라서 이 규칙은 `정산서 원문 자체가 ...제 + 권/화` 구조인지 확인하는 가드가 필요하다. 단순히 `정제키 + 제`가 S2에 있다는 이유만으로 자동 매핑하면 위험하다.

## 판단 절차 제안

1. `판매채널 + 정제키` 551개를 기준으로 본다.
2. 먼저 `same channel near-hit` 후보를 뽑는다.
   - 예: 빈 키 복구, 제목 말미 `제` 복구처럼 같은 채널 S2에 거의 확정 후보가 있는 경우.
3. `other channel only` 235개는 로직 후보에서 분리한다.
   - 귀축기담처럼 S2에는 있지만 해당 판매채널 지급정산이 없는 케이스가 많다.
   - 이 버킷은 로직 수정이 아니라 데이터/채널 기준 판단이다.
4. `no S2 key any channel` 315개는 행 수 순으로 2차 샘플링한다.
   - 상위 행 수 그룹부터 보면 효과가 크다.
   - 단, 큰 그룹이어도 S2 지급정산 자체가 없으면 로직을 건드리지 않는다.

## 분할 단위

추천 단위는 `50개 그룹`이다.

- 총 551개라서 약 12묶음.
- 한 묶음이 사람이 판단하기에 너무 작지도 크지도 않다.
- 각 묶음에는 `채널`, `현재 정제키`, `행 수`, `원문 예시`, `S2 같은 채널 후보`, `S2 다른 채널 후보`, `판정`을 둔다.

우선순위:

1. 즉시 로직 후보 10개 이하 별도 묶음
2. `other channel only` 235개를 50개씩 5묶음
3. `no S2 key any channel` 315개를 행 수 내림차순 50개씩 7묶음
