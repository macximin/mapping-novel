# 판매채널-지급정산 불일치 작업방

이 폴더는 S2의 `판매채널콘텐츠`와 `지급정산관리`가 서로 안 맞는 사례를 모으고 판정하기 위한 작업방입니다.

## 먼저 볼 파일

1. `01_공유폴더_정산서_샘플/파일명_자동감지_검토결과.csv`
   - 공유폴더 정산서 파일명이 현재 앱에서 자동감지되는지 확인한 표입니다.
   - `NO_S2_CHANNEL_IN_FILENAME`이면 파일명만으로는 S2 판매채널을 확정할 수 없습니다.

2. `03_S2_판매채널콘텐츠_증거/무툰_증거/무툰_판매채널-지급정산_불일치_증거.csv`
   - `판매채널콘텐츠`에는 있지만 `지급정산관리`에는 없는 실제 사례입니다.
   - `[정산정보없음]` 후보를 판단하는 증거로 봅니다.

3. `00_먼저_읽기/판정원칙.md`
   - 어떤 경우를 차단하고, 어떤 경우를 파일명 수정이나 앱 alias로 처리할지 정한 기준입니다.

## 폴더 구조

- `01_공유폴더_정산서_샘플`
  - 공유폴더에서 플랫폼별 최신 월 정산상세를 복사해 둔 곳입니다.
  - 37개 플랫폼, 51개 파일 기준입니다.

- `02_S2_지급정산관리_기준`
  - 정산 매핑의 기준 자료입니다.
  - 정산 전송용 판매코드는 여기 기준을 우선합니다.

- `03_S2_판매채널콘텐츠_증거`
  - `schn-ctns-search` 쪽 자료를 모으는 곳입니다.
  - 이 자료는 진단용입니다. 정산 매핑 기준으로 바로 쓰지 않습니다.

- `04_IPS_보조자료`
  - IPS 콘텐츠 보조 기준입니다.
  - 콘텐츠 존재 여부, 담당/장르 확인에 씁니다.

- `05_불일치_판정`
  - 실제 조치 대상을 넣는 곳입니다.
  - `[정산정보없음]`, 파일명 수정, 앱 alias, 수동확인으로 나눕니다.

- `06_공지_업무요청`
  - 다른 사람에게 요청하거나 공지할 문안 초안을 넣는 곳입니다.

## 핵심 결론

정산 매핑 기준은 `지급정산관리`입니다.

`판매채널콘텐츠`에 코드가 있어도 `지급정산관리`에 없으면, 정산정보 없는 판매코드일 수 있으므로 자동 연결하지 않습니다.
